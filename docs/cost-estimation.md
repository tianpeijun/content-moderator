# 商城评论内容审核系统 — AWS 部署成本评估

## 业务假设

| 参数 | 值 | 说明 |
|------|------|------|
| 日均审核量 | 2,000 条 | 需求文档定义 |
| 月均审核量 | 60,000 条 | 2,000 × 30 天 |
| 纯文本审核占比 | 80% | 48,000 条/月 |
| 图文混合审核占比 | 20% | 12,000 条/月 |
| 管理后台日活用户 | 3-5 人 | 产品经理 + 运营 |
| 批量测试频率 | 2 次/月 | 每次约 500 条 |
| 部署区域 | us-east-1 | 北弗吉尼亚 |

### 基于测试数据的 Token 估算

根据实际 API 测试结果（201 次调用，Bedrock 实际费用 $1.0521）：

| 指标 | 估算值 | 实际值 |
|------|--------|--------|
| 每次调用成本 | $0.0045 | $0.00523 |
| 差异 | — | +16.3% |

实际每次调用成本高于初始估算，主要因为动态标签指令增加了约 200 tokens 的 prompt 长度。以下月度成本使用实际校准值计算。

## 成本优化建议

### 短期优化

1. **切换备用模型为主模型**：对于大部分常规评论，Claude Haiku 4.5 已足够，可节省约 67% 的 Bedrock 费用
2. **启用 Bedrock 批量推理**：对非实时场景（如批量测试）使用批量推理，可节省 50% 的模型调用费用
3. **利用免费额度**：Lambda、SQS、CloudFront、Cognito 均在免费额度内

### 中期优化

4. **NAT Gateway 替代方案**：考虑使用 VPC Endpoint for Bedrock（$0.01/小时 + 数据费），可将 NAT Gateway 费用从 $32.63 降至约 $7.20/月
5. **RDS 预留实例**：1 年预留可节省约 30%，将 RDS 费用从 $17.28 降至约 $12.10/月
6. **Prompt Caching**：Anthropic 支持 prompt caching，对重复的规则模板部分可节省高达 90% 的输入 token 费用

### 长期优化

7. **Amazon Nova 模型**：对简单评论使用 Amazon Nova Lite（$0.06/$0.24 per 1M tokens），成本可降至 Sonnet 的 2%
8. **智能路由**：简单评论用低成本模型，复杂/可疑内容用高性能模型，可整体降低 50-70% 的 Bedrock 费用
