## 一、Amazon Bedrock（AI 模型调用）

| 项目 | 单价 |
|------|------|
| 输入 | $1.00 / 百万 tokens |
| 输出 | $5.00 / 百万 tokens |

### 月度成本计算（使用主模型 Claude Sonnet 4）

**纯文本审核（48,000 条/月）：**
- 输入 tokens：48,000 × 600 = 28,800,000 tokens
- 输出 tokens：48,000 × 150 = 7,200,000 tokens
- 输入成本：28.8M / 1M × $3.00 = **$86.40**
- 输出成本：7.2M / 1M × $15.00 = **$108.00**

**图文混合审核（12,000 条/月）：**
- 输入 tokens：12,000 × 800 = 9,600,000 tokens
- 输出 tokens：12,000 × 200 = 2,400,000 tokens
- 输入成本：9.6M / 1M × $3.00 = **$28.80**
- 输出成本：2.4M / 1M × $15.00 = **$36.00**

**批量测试（1,000 条/月）：**
- 输入成本：1,000 × 600 / 1M × $3.00 = **$1.80**
- 输出成本：1,000 × 150 / 1M × $15.00 = **$2.25**

**提示词预览/测试（约 200 次/月）：**
- 估算成本 ≈ **$1.00**

| Bedrock 子项 | 月成本 |
|-------------|--------|
| 纯文本审核 | $194.40 |
| 图文混合审核 | $64.80 |
| 批量测试 | $4.05 |
| 提示词测试 | $1.00 |
| **Bedrock 合计** | **$264.25** |

> 如果切换为 Claude Haiku 4.5 作为主模型，Bedrock 成本可降至约 **$88.08/月**（降低约 67%）。

## 成本优化建议

### 短期优化

1. **切换备用模型为主模型**：对于大部分常规评论，Claude Haiku 4.5 已足够，可节省约 67% 的 Bedrock 费用
2. **启用 Bedrock 批量推理**：对非实时场景（如批量测试）使用批量推理，可节省 50% 的模型调用费用
3. **利用免费额度**：Lambda、SQS、CloudFront、Cognito 均在免费额度内

### 中期优化

4. **NAT Gateway 替代方案**：考虑使用 VPC Endpoint for Bedrock（$0.01/小时 + 数据费），可将 NAT Gateway 费用从 $32.63 降至约 $7.20/月
5. **RDS 预留实例**：1 年预留可节省约 30%，将 RDS 费用从 $17.28 降至约 $12.10/月
6. **Prompt Caching**：Anthropic 支持 prompt caching，对重复的规则模板部分可节省高达 90% 的输入 token 费用

### 长期优化

7. **Amazon Nova 模型**：对简单评论使用 Amazon Nova Lite（$0.06/$0.24 per 1M tokens），成本可降至 Sonnet 的 2%
8. **智能路由**：简单评论用低成本模型，复杂/可疑内容用高性能模型，可整体降低 50-70% 的 Bedrock 费用
